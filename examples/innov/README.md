# p09 实验室目录 — 实验方案

## 实验目的

验证统一的实验室目录索引能否解决"东西太多整理费劲"的问题。

## 背景

日志 2026-06-01：

> "实验室里面产生的东西太多了，整理起来会很费劲"
> "未来的话就是说上平台要能够去整理海量的实案，要不然手动整理到什么时候"
> "以仓库为单位直接进行丢弃"

现在已经有 7 个实验室（game/human/org/exec/company/innov/default），每个里面有多个 PoC，没有统一的查看入口。需要一个目录把所有实验室汇总到一起。

## 实验假设

1. 一个轻量级的目录页面能清晰展示所有实验室的状态和内容，比翻 GitHub 组织页面更直观
2. 按领域分组 + 状态标记（active/archived/deprecated）比纯列表更好用
3. 目录本身也是 default 实验室的一个 PoC，数据内嵌在 HTML 中，无需后端

## 实验设计

### 阶段 1：数据结构定义

定义每个实验室的数据字段：

```json
{
  "id": "human",
  "name": "量潮人力资源实验室",
  "repo": "quanttide/quanttide-laboratory-of-human-resources",
  "domain": "人力资源",
  "status": "active",
  "pocCount": 1,
  "description": "招聘筛选、人才管理相关 PoC",
  "pocs": [
    { "id": "p01", "name": "招聘筛选网关", "status": "done" }
  ]
}
```

预填全部 7 个实验室和各自 PoC 的数据。

### 阶段 2：目录页面渲染

单 HTML 页面（零依赖），包含：

- **顶部统计**：实验室总数 / active 数 / PoC 总数
- **卡片网格**：每个实验室一张卡片
  - 名称 + 领域徽章
  - 状态标记（active=绿 / archived=灰 / deprecated=红）
  - PoC 列表（done/in_progress/pending）
  - 链接到 GitHub repo
- **筛选栏**：
  - 领域下拉
  - 状态切换（全部 / active / archived）
  - 关键词搜索

### 阶段 3：状态管理实验

模拟"以仓库为单位丢弃"的场景：

- 将某个实验室标记为 `deprecated`
- 确认目录页面自动隐藏（可通过 `?show=all` 参数显示）
- 确认其他实验室不受影响

### 阶段 4：CI 自动更新

编写一个 GitHub Actions 工作流，每周自动：

1. 检查各仓库的 `README.md` 提取描述
2. 检查各仓库的 `docs/poc/` 目录更新 PoC 数量
3. 生成新的 `data.json`
4. 提交 PR 更新目录页面

## 测试用例

| 用例 | 操作 | 预期 |
|------|------|------|
| 浏览目录 | 打开 index.html | 所有实验室以卡片展示 |
| 按领域筛选 | 选择"人力资源" | 只显示 human 实验室 |
| 搜索 | 输入"招聘" | 显示包含招聘关键字的实验室 |
| 标记废弃 | 修改 JSON 中 status 为 deprecated | 卡片默认隐藏，?show=all 显示 |
| 新增实验室 | 新增一条数据 | 目录自动包含新卡片 |

## 评价标准

| 指标 | 合格线 | 优秀线 |
|------|--------|--------|
| 实验室覆盖 | 5 个 | 全部 7 个 |
| 筛选响应 | < 500ms | < 200ms |
| 状态管理 | 手动改 JSON | CI 自动更新 |
| 移动端 | 可用 | 良好 |
