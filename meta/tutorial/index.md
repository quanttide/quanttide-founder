## 🧠 什么是创始人第二大脑

创始人第二大脑不是一个普通的文档仓库，而是一个**会生长的知识生命体**。

### 它的核心特征

1. **记忆分类框架**
   - 九宫格模型：按时间×类型分类知识
   - 五层体系：从论文到章程的完整链条

2. **自动化维护**
   - AI 辅助整理元数据
   - 自动记录和提炼思考过程

3. **人机协作**
   - 人类主导规则制定
   - AI 主导日常维护

## 🚀 如何使用创始人第二大脑

### 第一步：了解结构

#### 1. 元数据目录 (`meta/`)

这是第二大脑的"操作系统"：

```
meta/
├── bylaw/      # 工作章程（最高优先级）
│   ├── bylaw.md    # 元章程，定义人机分工
│   ├── journal.md  # 日志规范
│   └── profile.md  # Profile 维护章程
├── handbook/   # 工作手册（次高优先级）
│   └── profile.md  # Profile 维护经验
├── profile/    # 工作档案（中等优先级）
│   ├── repo.md     # 仓库结构
│   ├── release.md  # 版本管理
│   ├── memory.md   # 记忆建模
│   └── submodule.md # 子模块管理
├── journal/    # 工作日志（基础优先级）
│   └── [按日期命名]
├── brochure/   # 宣传册
│   └── index.md    # 项目宣传册
└── tutorial/   # 使用指南
    ├── index.md    # 本指南
    ├── git.md      # Git 使用最佳实践
    └── code/
        └── git.md  # Git 人机协作指南
```

#### 2. 子模块目录

每个子模块都是一个独立的知识库：

| 子模块 | 定位 | 用途 |
|--------|------|------|
| archive | 过去-事件类 | 工作归档 |
| journal | 现在-事件类 | 工作日志 |
| report | 现在-语义类 | 工作报告 |
| profile | 未来-事件类 | 工作档案 |
| essay | 未来-语义类 | 工作札记 |
| handbook | 程序型-习惯法 | 工作手册 |
| usercase | 程序型-判例法 | 工作案例 |
| paper | 程序型-权威法理 | 工作论文 |
| specification | 程序型-成文法 | 工程标准 |
| platform | 程序型-宪法 | 技术平台 |
| tutorial | 过去-语义类 | 工作教程 |

### 第二步：查找信息

#### 场景 3：查看最新状态

**包含内容**：

**去哪里找**：`meta/report/`

**包含内容**：

### 第三步：参与协作

#### 1. 提交改进意见

如果发现文档有问题或有改进建议：

1. 在对应仓库创建 Issue
2. 标签选择 `documentation`
3. 描述具体问题和建议

#### 2. 贡献内容

如果你想贡献内容：

1. 阅读 `CONTRIBUTING.md` 了解贡献流程
2. 遵循 `meta/bylaw/` 中的规范
3. 提交 Pull Request

#### 3. 使用 AGENTS.md

如果你是 AI Agent 或想了解如何与 AI 协作：

- 阅读 `AGENTS.md` 了解 AI 工作规范
- 理解人机分工原则
- 高效协作

## 💡 使用技巧

### 技巧 1：按优先级阅读

根据记忆分层模型，按优先级阅读：

1. **bylaw**（最高）：先了解规则
2. **handbook**（次高）：再学习经验
3. **profile**（中等）：然后看状态
4. **journal**（基础）：最后读日志

### 技巧 2：善用引用

文档之间通过引用连接：

- `IDENTITY.md` 引用 `profile/memory.md`
- `profile/release.md` 引用 GitHub Release
- `journal/` 按日期引用

**遇到引用 → 跳转查看 → 获得完整信息**

### 技巧 3：关注更新日志

定期查看：

- `meta/report/`：按日期查看最新日志
- `meta/profile/release.md`：查看版本更新
- GitHub Release：查看正式发布说明
