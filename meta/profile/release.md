## 版本约定

- 各子模块独立版本管理
- 主仓库版本通过 Git 标签管理
- Meta 目录由 AI 自动维护

## Release 流程规范

### 版本管理原则
- **版本标签**：使用 Git 标签管理版本（格式：0.1.0）
- **Release Notes**：遵循 Keep a Changelog 格式
- **元数据更新**：发布后更新 `meta/profile/release.md` 和 `meta/IDENTITY.md`

### 版本控制原则
- **正式发布**：使用正式 Release 而非草稿
- **版本规范**：遵循语义化版本控制规范
- **清理维护**：定期清理不必要的 Release
