## 语义记忆

### 记忆建模方法

来源：`quanttide-specification-of-business-entity/think/memory/`

记忆系统分为两大类：

#### 陈述性记忆（Declarative / Explicit Memory）

三种基本类型：

| 类型 | 说明 | 例子 |
|------|------|------|
| **事件记忆（Episodic）** | 个人经历的、发生在特定时空的具体事件，带有情境化细节（时间、地点、参与者、情感） | "昨天下午在书房尝试用 opencode 开发管理后台，遇到困难，感到很挫败。" |
| **语义记忆（Semantic）** | 客观的、去情境化的知识，包括事实、概念、规则，不依赖特定时空背景 | "opencode 适合编辑静态文本文件，但对于需要精密细节的代码编写，目前能力有限。" |
| **自我记忆（Self）** | 与自我认知、身份、价值观、情感反思相关的记忆，记录"我是谁"和成长变化 | "我一直期待自己成为顶尖程序员，但今天遇到的困难让我意识到，我的执行力与预期还有差距。" |

#### 程序性记忆（Procedural Memory）

关于"怎么做"的记忆，记录技能、流程、操作方法。

### 事件记忆建模

事件记忆的结构化字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | 是 | UUID，全局唯一标识符 |
| `title` | string | 是 | 事件的简短标题 |
| `description` | string | 是 | 详细描述，可包含原始想法、细节 |
| `tense` | enum | 是 | 时间属性：`past`（回顾）、`present`（实时）、`future`（规划） |
| `type` | enum | 是 | 事件类型：`decision`（决策）、`plan`（计划）、`report`（汇报）、`evaluation`（评估）、`retrospective`（回顾） |
