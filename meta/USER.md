# 用户

## 谁在使用

**主要用户**：量潮创始人

**次要用户**：量潮科技

## 用户画像

### 创始人

**身份**：创业者、思考者、写作者

**核心特质**：
- 相信知识积累的价值
- 重视系统性思考
- 喜欢用工具放大自己的能力
- 乐于分享和开源

**日常工作**：
- 思考：战略、产品、技术方向
- 写作：博客、文档、代码
- 学习：新技术、新方法、新领域
- 连接：与团队、用户、伙伴沟通

**知识管理痛点**：
- 知识碎片化，难以形成系统
- 日记归档太复杂，难以坚持
- 思考过程丢失，难以追溯
- 团队知识难以沉淀和传承

## 用户目标

### 短期目标

1. **建立第二大脑**：让所有思考、学习、工作都有记录
2. **形成知识闭环**：日记 → 档案 → 教程 → 案例的流动
3. **提升写作效率**：通过素材库和模板快速产出
4. **保持思考记录**：不丢失任何有价值的想法

### 长期目标

1. **知识体系化**：形成完整的个人知识图谱
2. **经验可传承**：让团队成员能够理解创始人的思考逻辑
3. **开源影响力**：通过公开知识建立行业影响力
4. **持续进化**：知识系统能够随着认知升级而自动演进

## 使用场景

### 日常场景

| 场景 | 使用模块 | 频率 |
|------|----------|------|
| 记录当天工作 | journal | 每日 |
| 深度思考问题 | essay | 每周多次 |
| 归档重要资料 | archive | 按需 |
| 编写技术文档 | handbook | 按需 |
| 学习新知识 | tutorial | 按需 |
| 制定工程标准 | specification | 按需 |
| 总结经验教训 | usercase | 按需 |
| 规划未来方向 | profile | 按需 |
| 产出研究报告 | report | 每周 |
| 发表原创文章 | paper | 每月 |

### 典型工作流

1. **日常记录**：journal → 当天的工作、思考、决策
2. **周度总结**：journal → report → 提炼本周重点
3. **深度思考**：essay → 系统性分析和写作
4. **知识沉淀**：essay → handbook → 形成可复用的方法
5. **经验归档**：journal → archive → 保留原始事件记录

## 用户期望

### 对系统的期望

- **简单易用**：记录不应该成为负担
- **结构清晰**：知道东西应该放在哪里
- **快速检索**：能随时找到需要的信息
- **自动流转**：日记自动变成档案，案例自动变成教程
- **AI 辅助**：AI 能帮助整理、提炼、关联知识

### 对 AI 的期望

- **最小干预**：不主动打扰，只在需要时出现
- **理解语境**：理解我在做什么，为什么这么做
- **提供价值**：给出有意义的建议，不是废话
- **保持一致**：遵守我的工作习惯和偏好

## 使用原则

### 核心原则

1. **记录为王**：记录比整理重要，先记录再说
2. **结构化思考**：用框架思考，用模板记录
3. **定期回顾**：每周回顾，每月归档
4. **持续迭代**：不断优化工作流和模板

### 注意事项

- 不要追求完美，记录本身就有价值
- 不要过度分类，太复杂的系统会坚持不下去
- 不要完全依赖 AI，保持人的判断和创造力
- 不要闭门造车，知识要流动才有价值

## 使用反馈

- 日常问题：`journal/meta/`
- 重大决策：`journal/deliberation/`
- 系统改进：`meta/report/`
- 功能需求：`handbook/product/`
