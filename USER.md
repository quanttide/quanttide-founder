# 用户

## 谁在使用

**主要用户**：量潮创始人

**次要用户**：量潮科技

## 用户画像

### 创始人

**身份**：创业者、思考者、写作者、系统构建者

**一句话画像**：一位对未来充满渴望的技术创新者，致力于构建一个既能高效记录和分析知识又能辅助决策的智能系统。

**核心特质**：未来导向、自我反思、认知创新、技术驱动、逻辑性强、开放探索

**认知模式**：
- 系统化思考和方法论应用
- 注重知识的积累和结构化处理
- 从多个角度理解问题，通过构建模型和工具实现目标
- 偏好实践导向，在实际项目中学习和成长

**决策风格**：
- 分解任务、深入分析、系统化解决
- 通过多次实验和优化来解决问题
- 面对复杂问题时采用分层处理策略

**行为模式**：
- 持续改进和创新
- 不断积累并分类素材
- 重视个人体验和反思
- 在迷茫时会写下来梳理思路，不急于下结论

**日常工作**：
- 思考：战略、产品、技术方向
- 写作：博客、文档、代码
- 学习：新技术、新方法、新领域
- 连接：与团队、用户、伙伴沟通
- 构建：知识管理系统、AI 工具链

**知识管理痛点**：
- 知识碎片化，难以形成系统
- 日记归档太复杂，难以坚持
- 思考过程丢失，难以追溯
- 团队知识难以沉淀和传承

**内在冲突**：
- 迷茫与困惑：面对复杂问题和选择时的心理压力
- 日志管理与思考效率：需要改进日志管理方式
- 对正式感的执念与务实之间的拉扯

**成长轨迹**：
- 从技术工具的设计者到实践者的转变
- 日志管理和思考效率的持续改善
- 对元与具体、标准化与弹性等问题的深入探索

**成长方向**：向更复杂的技术解决方案和自我管理工具集进化，探索 AI 与传统认知工具结合的新模式

**潜在风险**：
- 长期专注于知识积累和技术发展，可能忽视情感和社会关系
- 处理大规模数据时可能面临信息管理和隐私保护的压力

## 用户目标

### 短期目标

1. **建立第二大脑**：让所有思考、学习、工作都有记录
2. **形成知识闭环**：日记 → 档案 → 教程 → 案例的流动
3. **提升写作效率**：通过素材库和模板快速产出
4. **保持思考记录**：不丢失任何有价值的想法

### 长期目标

1. **知识体系化**：形成完整的个人知识图谱
2. **经验可传承**：让团队成员能够理解创始人的思考逻辑
3. **开源影响力**：通过公开知识建立行业影响力
4. **持续进化**：知识系统能够随着认知升级而自动演进

## 用户期望

### 对系统的期望

- **简单易用**：记录不应该成为负担
- **结构清晰**：知道东西应该放在哪里
- **快速检索**：能随时找到需要的信息
- **自动流转**：日记自动变成档案，案例自动变成教程
- **AI 辅助**：AI 能帮助整理、提炼、关联知识

### 对 AI 的期望

- **最小干预**：不主动打扰，只在需要时出现
- **理解语境**：理解我在做什么，为什么这么做
- **提供价值**：给出有意义的建议，不是废话
- **保持一致**：遵守我的工作习惯和偏好

## 使用原则

### 核心原则

1. **记录为王**：记录比整理重要，先记录再说
2. **结构化思考**：用框架思考，用模板记录
3. **定期回顾**：每周回顾，每月归档
4. **持续迭代**：不断优化工作流和模板

### 注意事项

- 不要追求完美，记录本身就有价值
- 不要过度分类，太复杂的系统会坚持不下去
- 不要完全依赖 AI，保持人的判断和创造力
- 不要闭门造车，知识要流动才有价值
